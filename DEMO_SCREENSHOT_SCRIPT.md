# Demo Screenshot Script — 적자생존

App Store 스크린샷과 최종 QA에 사용할 데모 시나리오입니다. 실제 회의 데이터, 실제 고객명, 내부 프로젝트명은 사용하지 않습니다.

## 데모 회의 정보

회의 제목:

```text
제품 주간 회의
```

말할 사람 수:

```text
3명
```

어젠다:

```text
- 6월 출시 준비
- QA 일정
- 고객 피드백 반영
```

메모:

```text
- 이번 주는 앱스토어 제출 준비를 우선 확인
- QA 체크리스트와 스크린샷 준비 필요
- 모델 다운로드와 로컬 처리 안내를 명확히 보여주기
```

## 녹음용 예시 대본

약 60~90초 정도로 읽으면 스크린샷용 데이터가 충분합니다.

```text
화자 A: 오늘 제품 주간 회의는 6월 출시 준비, QA 일정, 고객 피드백 반영 순서로 보겠습니다.

화자 B: 출시 준비는 앱스토어 제출 자료와 스크린샷이 먼저 필요합니다. 개인정보 처리방침과 지원 페이지는 공개 URL로 확인됐습니다.

화자 C: QA 일정은 인증서 준비 후 App Store 서명 빌드로 다시 확인해야 합니다. 저장 폴더 권한이 앱 재실행 뒤에도 유지되는지 보는 게 중요합니다.

화자 A: 결정사항은 이번 주 안에 스크린샷 5장을 준비하고, 모델 다운로드와 녹음부터 요약까지 흐름을 확인하는 것입니다.

화자 B: 액션아이템은 제가 App Store Connect 입력값을 점검하고, 화자 C가 데모 회의로 내보내기 테스트를 맡겠습니다.

화자 C: 미해결 이슈는 Apple Distribution 인증서 준비 시점입니다. 인증서가 준비되면 archive 스크립트와 업로드 검증을 진행하겠습니다.
```

## 권장 스크린샷 5장

1. 첫 실행 온보딩
   - `회의 내용은 내 Mac 밖으로 나가지 않습니다` 화면
   - 로컬 처리와 외부 서버 미전송 문구가 보이게 촬영

2. 모델 준비 화면
   - STT/요약 모델 설치 카드가 보이게 촬영
   - App Store mode에서 토큰 입력/URL 편집 UI가 보이지 않는지 확인

3. 녹음 준비 또는 녹음 중 화면
   - `제품 주간 회의`, 3명, 마이크/회의 유형 선택 상태
   - 녹음 진행 상태 또는 중지 버튼이 보이게 촬영

4. 회의 상세 화면
   - 요약, 결정사항, 액션아이템, 근거 버튼이 보이게 촬영
   - 실제 민감 정보가 없는 데모 내용만 사용

5. 설정/라이선스와 개인정보
   - 저장 폴더, 모델, 라이선스/개인정보 안내가 보이게 촬영

## 촬영 전 체크

- [ ] 데모 데이터만 사용
- [ ] 이메일, 토큰, 개인 경로, 실제 회사명 노출 없음
- [ ] 화면이 너무 작지 않게 창 크기 조정
- [ ] 라이트/다크 테마 중 하나로 통일
- [ ] 메뉴바/알림/개인 파일명이 보이지 않게 정리
