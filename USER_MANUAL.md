## 1. 이 앱은 뭘 해주나요?

**대면 한국어 회의를 녹음하면, 내 Mac에서 AI가 전사본과 요약을 만들어줍니다.**

- ✅ **완전 온디바이스**: 녹음·전사·요약이 전부 Mac 안에서 처리됩니다. 외부 서버로 전송 안 됩니다.
- ✅ **한국어 전용 튜닝**: STT는 Whisper large-v3-turbo, 요약은 Gemma 4 E2B를 한국어 회의용으로 파라미터 조정했습니다.
- ✅ **비용 0원**: API 호출 비용이 없습니다. 모델 다운로드 한 번이면 끝.
- ⚠️ **오프라인 작동**: 인터넷이 없어도 동작합니다(모델만 미리 받아두면).

**사용 시나리오 예시**
- 스탠드업/기획/회고/1:1 회의 자동 기록
- 회의 직후 참석자들에게 요약 공유
- 회의 내용을 나중에 검색해서 다시 찾기

## 2. 설치하기

### 시스템 요구사항
- **OS**: macOS 13 이상
- **CPU**: Apple Silicon(M1 이상) 강력 권장 — Intel Mac은 속도가 5~10배 느립니다
- **메모리**: 최소 16GB 권장 (Gemma 요약 단계 피크 ~9GB 사용. 8GB Mac은 타 앱 모두 종료 필수)
- **디스크**: 앱 ~30MB + 모델 ~8GB = **약 9GB** 여유 공간

### 설치 단계

1. 팀에서 받은 `적자생존_v1.9.1.dmg` 파일을 더블클릭
2. `적자생존.app`을 **Applications 폴더**로 드래그
3. DMG 창에 있는 **"⚡ 처음 여기를 먼저 실행하세요"** 를 더블클릭
4. Mac 비밀번호 입력 → 앱이 자동 실행됩니다

> 왜 도우미 앱이 필요한가요? macOS Gatekeeper가 미서명 앱을 차단하기 때문에, 도우미가 `xattr -rd com.apple.quarantine` 명령으로 격리 속성을 제거해줍니다. 터미널을 열 필요 없이 한 번만 실행하면 됩니다.

## 3. 첫 실행 & 모델 파일

앱을 처음 실행하면 **모델 파일이 없다는 안내**가 나옵니다. 팀에서 전달받은 두 개의 GGUF 파일을 아래 경로에 넣으세요:

```
~/Library/Application Support/LocalMinutes/models/
```

필요한 파일:

| 파일 | 용도 | 크기 |
|---|---|---|
| `whisper-large-v3-turbo-q8_0.gguf` | 한국어 음성 → 텍스트 | ~1.5GB |
| `gemma-3n-E2B-it-Q8_0.gguf` | 전사본 → 요약 | ~6GB |

**빠른 복사 방법 (Finder)**
1. Finder에서 `Shift + Cmd + G` → 위 경로 붙여넣기 → Enter
2. 받은 파일 두 개를 드래그해서 그 폴더에 놓기
3. 앱 재시작 — 이제 녹음·요약 가능

```

│  사이드바    │  회의 목록     │  회의 상세       │

│ • 🏠 전체    │  검색창         │  제목 / 날짜     │
│ • 📁 그룹1   │  회의 카드들    │  요약            │

│ • ⚙ 설정     │                │  액션 아이템     │
│ • ➕ 녹음    │                │                  │
```

**사이드바 주요 버튼**
- 🎙 **새 녹음**: 녹음 화면으로 진입

- 🔍 **검색**: 제목·메모·태그·요약·전사본 전체 검색
- 📊 **통계**: 회의 길이·시간대·키워드 분석
- ⚙ **설정**: 테마, 요약 템플릿, 단어집

## 5. 회의 녹음 & 요약하기

### 새 녹음 시작
1. 사이드바 **🎙 새 녹음** 클릭
2. 제목 입력 (예: "2026-04-21 주간회의")
3. *(선택)* 참석자 이름 추가 — 나중에 요약에 반영됨
4. *(선택)* 요약 템플릿 선택 (아래 [6장](#6-요약-템플릿-고르기) 참고)
5. 마이크 기기 선택 (내장/외장 USB 마이크 등)
6. **녹음 시작** 버튼

### 녹음 중에 할 수 있는 것
- ⏸ **일시정지 / 재개**
- 📝 **메모 입력**: 중요한 포인트를 실시간으로 적어두면 요약 시 AI가 참고합니다
- 🛑 **녹음 종료** → 자동으로 전사·요약 파이프라인 시작

### 자동 처리 단계
녹음 종료 후 다음이 **순차적으로** 돌아갑니다 (메모리 예산 때문에 동시 실행 불가):

```
1. Whisper 모델 로드         (~2GB 메모리)
2. 한국어 STT 전사           (10분 녹음 → 약 1~2분)
3. Whisper 해제
4. Gemma 모델 로드           (~6GB 메모리)
5. 회의 요약 생성            (약 30초~2분)
6. Gemma 해제 · DB 저장
```

M3 Mac 기준 **10분 녹음 → 총 3~4분 처리**.

처리 중에 앱을 닫지 마세요. 진행 상황은 화면에 단계별로 표시됩니다.

## 6. 요약 템플릿 고르기

같은 회의 내용도 **어떤 관점으로 정리할지**에 따라 요약이 달라집니다. 5개의 프리셋 + 커스텀이 있습니다:

| 템플릿 | 용도 | 특징 |
|---|---|---|
| 🟢 **일반 회의** | 범용 회의록 | 논의·결정·액션을 고르게 정리 |
| 🟡 **스탠드업** | 데일리 동기화 | "어제/오늘/블로커" 구조로 참석자별 정리 |
| 🔵 **기획 회의** | 킥오프·스코프 정의 | 목표·범위·일정·리스크 구분 |
| 🟣 **회고** | 스프린트 회고 | Keep / Problem / Try 접두사로 분류 |
| 🟠 **인터뷰/1:1** | 1:1, 사용자 인터뷰 | Q:/A: 쌍으로 정리 |
| ⚪ **커스텀** | 직접 지시 작성 | 설정에서 프롬프트 직접 편집 |

### 어디서 선택하나요?
- **녹음 시작 화면**: 드롭다운에서 이번 회의에만 적용할 템플릿 선택
- **회의 상세 → "다시 요약" 버튼**: 이미 끝난 회의를 다른 템플릿으로 재요약
- **설정 → 요약 템플릿**: 전역 기본값 지정 (회의별 오버라이드가 없을 때 적용)

### 어떻게 다른가요?
같은 전사본이라도 템플릿마다 결과가 달라집니다. 예시:

**전사본**: "민수: 어제 로그인 API 붙였고, 오늘 세션 만료 이슈. 블로커는 Redis 권한."

- **일반**: `keyDiscussions: ["로그인 API", "세션 이슈"]`
- **스탠드업**: `keyDiscussions: ["민수 — 어제: 로그인 API / 오늘: 세션 이슈"]`, `openQuestions: ["민수 블로커: Redis 권한"]`

## 7. 회의 기록 활용

회의가 끝나면 **회의 상세 화면**에서 다음을 할 수 있습니다:

### 요약 섹션
- **수정**: 요약 텍스트를 직접 편집 가능. 편집하면 이전 버전은 **이력으로 자동 저장** (되돌리기 가능)
- **📜 이력 보기**: 과거 버전들 조회 + 복원
- **✨ 다시 요약**: 다른 템플릿으로 재실행 (기존 요약은 이력에 보관)

### 액션 아이템
- 체크박스로 **완료 표시**
- ➕ 버튼으로 **수동 추가** (담당자·기한 입력)
- 항목 편집/삭제

### 전사본
- 문단별 타임스탬프 `[00:15 → 00:42]`
- **구간 재생**: 문단 클릭 시 해당 시점 오디오 재생 (audioFilePath 있을 때)
- 전체 복사 가능

### 메모
- 녹음 중 적은 메모 + 사후 추가 메모
- 요약 재실행 시 AI가 참고

### 단어 자동 추출
- 전사본에서 **자주 등장한 고유명사**를 추출해 단어집에 등록 제안

## 8. 검색 · 그룹 · 태그

### 🔍 전문 검색
사이드바 상단 검색창에 단어 입력하면 **5개 필드**에서 찾아줍니다:
- 제목 · 메모 · 태그 · **요약 전 필드** · **전사본 본문**

결과 카드에는:
- 📌 **어디서 찾았는지** 배지 (제목/요약/전사본 등)
- **하이라이트 스니펫** (±32자 주변 맥락)
- AND 검색 지원: `"Redis 권한"` 두 단어 모두 있는 회의만

### 📁 그룹
- 사이드바 **"+ 새 그룹"** → 프로젝트명 입력
- 회의 카드를 그룹으로 드래그 or 상세 화면에서 그룹 지정
- 그룹별로 필터링해서 보기

### 🏷 태그
- 회의 상세 화면 하단의 **태그 칩** 영역
- 자주 쓴 태그는 자동 제안
- 예: `기획`, `1on1`, `리뷰`, `긴급`
- 통계 화면에서 **태그별 회의 수/평균 시간** 분석

## 9. 단어집 (전문 용어 등록)

**단어집은 두 군데에서 활용됩니다:**

### (1) STT 정확도 향상 (v1.9.1~)
녹음 시작 전에 등록된 단어들이 Whisper의 `initial_prompt`에 주입되어, **회사 고유명사를 정확히 받아적게** 합니다.

> 예: "넷마블", "Isar", "Gemma", "Riverpod" 같은 용어를 등록하면 일반 Whisper가 "네트 마블", "이자" 등으로 잘못 받아쓰던 것을 올바르게 인식합니다.

### (2) 요약 시 용어 해석
요약 단계에서 Gemma가 **단어집의 정의**를 참고해 용어를 올바른 의미로 해석합니다.

### 등록 방법
- **설정 → 단어집** 메뉴에서 수동 추가
- 회의 상세의 **"용어 자동 추출"** 버튼으로 후보 제안받기
- 입력 필드: `용어` + `설명`

💡 **팁**: 팀/프로젝트 고유명사 10~20개만 넣어도 인식률이 눈에 띄게 개선됩니다.

## 13. 자주 묻는 질문

### Q. 인터넷 없어도 되나요?
**A.** 네. 모델 파일만 미리 받아두면 비행기/보안 회의실에서도 동작합니다.

### Q. 녹음 파일은 어디 저장되나요?
**A.** `~/Library/Application Support/LocalMinutes/recordings/` 에 WAV로 저장. 회의 삭제 시 함께 삭제됩니다.

### Q. 회의 데이터가 외부로 전송되나요?
**A.** 아니요. 전사·요약 모두 로컬 모델로 처리됩니다. 네트워크 요청이 0건임을 Little Snitch 등으로 확인 가능합니다.

### Q. 한 회의를 다른 템플릿으로 다시 요약할 수 있나요?
**A.** 네. 회의 상세 → **"✨ 다시 요약"** → 템플릿 선택. 이전 요약은 자동으로 이력에 저장됩니다.

### Q. 요약이 이상해요. 어떻게 개선하나요?
**A.** 다음을 시도해보세요:
1. **메모 추가**: 중요한 맥락·결정을 메모에 적으면 요약 품질이 올라갑니다
2. **단어집 등록**: 고유명사가 틀리게 전사됐다면 단어집에 추가 후 STT 재실행
3. **템플릿 변경**: 회의 성격에 맞는 템플릿 선택 (스탠드업 → 스탠드업 템플릿)
4. **커스텀 프롬프트**: 설정에서 원하는 출력 형식 직접 지시

### Q. 녹음 품질이 떨어져요.
**A.** 외장 USB 마이크 사용 권장. 회의실 중앙에 놓거나, 화자 가까이 배치. 내장 마이크는 2m 이상 떨어지면 정확도 급락합니다.

### Q. M1 Mac에서도 되나요?
**A.** 네. M1/M2/M3 모두 동작. M3가 가장 빠릅니다.

### Q. Intel Mac은?
**A.** 이론상 되지만 5~10배 느리고 메모리 압박이 큽니다. 권장하지 않습니다.

## 14. 문제 해결

### 앱이 "손상되었다"며 안 열립니다
- DMG 안의 **"⚡ 처음 여기를 먼저 실행하세요"** 를 실행했는지 확인
- 안 됐다면 터미널에서: `xattr -rd com.apple.quarantine /Applications/적자생존.app`

### "Whisper 모델을 찾을 수 없습니다"
- `~/Library/Application Support/LocalMinutes/models/` 경로 존재 확인
- 파일명이 정확히 `whisper-large-v3-turbo-q8_0.gguf` 인지 확인 (대소문자, 하이픈)

### 요약이 중간에 잘립니다
- Gemma 컨텍스트(4096 토큰)를 초과한 경우
- 매우 긴 회의(90분+)는 요약이 잘릴 수 있음
- 대응: 회의를 구간별로 나눠 녹음하거나, 전사본을 수동 분할 후 재요약

### 전사에 환각(말 안 한 내용)이 섞입니다
- v1.9.1에서 `temperature=0`, `logprob_thold=-0.8` 조정으로 크게 완화됨
- 여전히 심하면 설정에서 STT 언어가 `ko`인지 확인
- 음질이 안 좋은 구간에서 주로 발생 → 마이크 위치 조정

### 처리 중 앱이 멎은 것 같아요
- Whisper/Gemma 로드에 30초~1분 걸립니다. 진행 단계 문구가 바뀌고 있으면 정상
- 활성상태보기(Activity Monitor)로 CPU 사용률 확인 — 100%+면 추론 중

### 메모리 부족 경고
- Gemma 요약 피크: ~9GB (모델 ~8GB + KV 캐시 ~1GB @ 8K context)
- 브라우저·Xcode·Docker 등 다른 무거운 앱 종료
- 16GB Mac에선 거의 문제없음, 8GB Mac은 스왑 발생 가능성 있음

### DB를 초기화하고 싶어요
- 앱 종료 후 다음 폴더 삭제:
  `~/Library/Application Support/LocalMinutes/`
- 주의: **모든 회의 기록이 사라집니다**. 중요한 데이터는 먼저 PDF/TXT로 내보내기
